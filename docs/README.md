# CAIAO 知识库 — 完整参考

> CAIAO 体系包含两部分：**框架代码**（`caiao/` + `caiao_lightweight/`）和**知识库文档**（本目录）。
> 框架代码是可执行的实体——安装或复制即用。知识库是设计原理、操作手册和参考指南。
> **这是 CAIAO 范式的权威参考文档。**

## 框架代码

| 路径 | 类型 | 获取方式 |
|------|------|---------|
| `caiao/` | MCP SDK 完整实现 | `pip install "git+https://github.com/LaobaiAi/CAIAO.git#subdirectory=caiao"` 或 `caiao init my-project` |
| `caiao_lightweight/` | 零依赖轻量实现 | 复制 `server.py` + `hub.py` + `subprocess.py` 到项目 |

## 关键洞察

### CAIAO 的两种实现路径：不是竞争，是分层

```
         ┌──────────────────────────────┐
         │     CAIAO 协议（概念层）     │
         │  list_tools / call_tool      │
         │  Server 独立 / 合并不修改     │
         └──────────┬───────────────────┘
                    │
        ┌───────────┴───────────┐
        │                       │
  ┌─────▼──────┐        ┌──────▼──────┐
  │ MCP SDK   │        │ 轻量 CAIAO  │
  │ 实现       │        │ 实现        │
  │           │        │             │
  │ mcp>=1.0  │        │ 纯 Python  │
  │ 异步       │        │ 同步        │
  │ 完整生命周期│       │ 零依赖      │
  │ AI 生态互通│        │ 开发效率优先 │
  └───────────┘        └─────────────┘
  生产级平台             开发框架/独立工具
```

**核心认知**：轻量实现不是 MCP SDK 实现的「简化版」或「过渡方案」——它是独立的架构选择。CAIAO 是开发框架（`@tool` 注册、自动发现、统一调用），MCP 是通信协议（AI 生态互通）。两者解决不同层次的问题。

### 选择指南速查

| 场景 | 推荐 |
|------|------|
| 新项目快速原型、单人开发 | 轻量实现 |
| 需要被 Claude Desktop 等外部 AI 调用 | MCP SDK 实现 |
| 多重量级求解器需进程隔离 | MCP SDK 实现 |
| 受限 Python 环境（Abaqus/Blender 内嵌） | 轻量实现 |
| 生产环境需健康监控和指标 | MCP SDK 实现 |
| 独立 CLI 工具 | 轻量实现 |

## 使用建议

1. **新项目想用 CAIAO**：读「01-core-concepts」→「10-operations-manual/新项目CAIAO化启动指南」→「02-two-implementations/适用场景对比」选择实现路径

3. **吸收外部项目**：严格按「10-operations-manual/项目吸收完整操作指南」五步法执行
4. **贡献 CAIAO Server**：读「05-dev-guides」→ 参考「10-operations-manual/Server合并完整操作指南」→「04-caiao.yaml 清单系统」
5. **从旧系统迁移**：读「06-Iron-Fall 参考」→「05-dev-guides/蒸馏方法论」
6. **做代码审查**：按「10-operations-manual/Simplify审查操作指南」四维度执行
7. **调试和测试**：参考「10-operations-manual/Server测试与调试指南」
8. **评估 CAIAO 成熟度**：读「09-critique-and-reflection」了解已知问题和改进方向
9. **规划架构演进**：读「09-critique-and-reflection/反思与改进方向」+「10-operations-manual/Blender-Daemon架构实施指南」+「08-evolution-history」
