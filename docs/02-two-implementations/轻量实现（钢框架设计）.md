# 轻量 CAIAO 实现模式

## 框架代码

轻量路径的框架代码位于 `caiao_lightweight/` 目录。三个文件，零外部依赖：

| 文件 | 行数 | 职责 |
|------|:---:|------|
| `server.py` | ~110 | `CAIAOServer` 基类 + `@tool` 装饰器 + stdin/CLI 双模式 |
| `hub.py` | ~100 | 同步 Hub（进程内直调 + 子进程管理） |
| `subprocess.py` | ~90 | JSON-line 协议子进程管理器 |

使用方式：复制这三个文件到项目目录即可。完整说明见 `caiao_lightweight/README.md`。

## 概述

这是 CAIAO 的**零依赖实现模式**，不依赖 MCP SDK。核心仅三个文件——Server 基类、Hub 调度中心、子进程管理器——复制即用。

适用于：
- 独立项目，不想引入 MCP SDK 重依赖
- 快速原型开发
- 学习和理解 CAIAO 核心概念

## 核心组件

### CAIAOServer 基类（约 110 行）

```
class CAIAOServer:
    # 类级元信息（子类覆盖）
    name: str = ""
    description: str = ""
    version: str = "0.1.0"

    def list_tools(self) -> list[dict]    # 声明能力
    def call_tool(self, name, args)       # 执行工具，返回 dict
    def get_metadata(self) -> dict        # 返回元数据
    def run_stdio(self)                   # stdin JSON-line 循环（子进程模式）
    def run_cli(self, args)               # CLI 调试入口
```

### @tool 装饰器

最简工具注册机制——装饰器不带参数，工具名自动取方法名，描述取自 docstring：

```
class MyServer(CAIAOServer):
    @tool
    def generate_frame(self, args: dict) -> dict:
        """根据网格参数生成框架结构模型。"""
        ...
        return {"nodes": [...], "elements": [...]}
```

装饰器在方法上设置 `_is_caiao_tool = True` 标记。基类 `__init__` 中通过反射自动扫描所有带此标记的方法并注册为工具。

### Hub（Hub 调度中心，约四百行）

轻量调度中心，支持两种运行模式：

#### in_process 模式
主进程内直调，适用于编排型 Server。Hub 自动扫描 Server 目录，实例化所有非子进程 `CAIAOServer` 子类并注册其工具。

#### subprocess 模式
独立子进程，通过 stdin/stdout JSON 行协议通信，适用于计算型 Server。使用 `SubprocessManager`（约百行）管理生命周期：惰性 spawn → JSON 请求/响应 → 优雅终止（SIGTERM → 5s → SIGKILL）。

## SubprocessManager 通信协议

轻量 JSON-RPC，每行一个完整 JSON：

请求：
```
{"method": "list_tools", "params": {}, "id": 1}
{"method": "call_tool", "params": {"tool_name": "run_analysis", "input": {...}}, "id": 2}
```

响应：
```
{"id": 1, "result": [{"name": "run_analysis", ...}]}
{"id": 2, "result": {"displacements": [...]}}
```

## 与 MCP SDK 实现的关系

轻量实现的所有 Server 接口（`list_tools` / `call_tool`）已与 MCP 协议对齐。`run_stdio_loop()` 中的手写 JSON-RPC 循环可以被 MCP SDK 的 `stdio_server()` 替换（128 行 → 10 行），业务逻辑零改动。

轻量实现是「你现在就需要的开发框架」，MCP SDK 实现是「完整生态平台」。两者共享同一套 CAIAO 设计哲学和 Server 模式。
