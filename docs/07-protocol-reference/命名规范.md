# CAIAO 命名规范

## 总则

CAIAO 是项目自有命名层，构建于 MCP SDK 之上。命名规范区分项目层（CAIAO 大写）和底层 SDK（保持原样）。

## 规范表

| 上下文 | 规范 | 示例 |
|--------|------|------|
| 类名 | `CAIAO` + PascalCase | `CAIAOClientHub` |
| 常量 | `CAIAO_` + UPPER_SNAKE | `CAIAO_SERVERS_DIR` |
| 文件名 | `caiao_` + 小写 | `hub.py`, `caiao/discovery.py` |
| 目录名 | `servers/` | `servers/anastruct_server/` |
| 配置中的 Server 名 | 小写 snake_case | commit scope `caiao` |
| SDK 导入 | 保持 `from mcp.server import Server` | 外部包，不重命名 |
| 工具名 | `snake_case` | `quick_analysis`, `analyze_frame` |
| 返回格式 | JSON in `TextContent` | `json.dumps({"status": "ok"})` |

## Server 命名

### 目录名
```
servers/<name>/
```
- 使用 snake_case
- 描述 Server 功能或引擎
- 示例：`anastruct_server`、`blender_environment_server`、`abaqus_session_server`

### 清单中的名称
- `caiao.yaml` 中 `name` 字段与目录名一致
- 在该 Server 的上下文内唯一

## 工具命名

- snake_case
- 动词开头：`generate_frame`、`analyze_structure`、`select_critical_element`
- 清晰描述做什么（不用缩写）
- 全局唯一（在同一 Hub 下不冲突）

## 管道/合并 Server 命名

- 描述管线做什么
- 示例：`quick_analysis`、`full_analysis_3d`、`run_full_pipeline`

## 基础设施 Server 命名

- `*_environment_server` 后缀
- 示例：`blender_environment_server`、`abaqus_environment_server`

## 禁止的命名

- 不要用 MCP 前缀（`mcp_*`）——这是项目层，不是协议层
- 不要用缩写（`gen_frm`）——AI 需要清晰语义
- 不要用驼峰命名文件（`caiaoHub.py`）——统一小写+下划线
- 不要用中文命名文件或工具名——代码用英文
