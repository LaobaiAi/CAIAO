# CAIAO 契约规则

## 核心契约

每个 CAIAO Server 必须实现三个接口：

### list_tools()

声明 Server 能力。返回工具描述列表：

```json
[
    {
        "name": "tool_name",
        "description": "人类可读描述，AI 据此理解功能",
        "inputSchema": {
            "type": "object",
            "required": ["param1"],
            "properties": {
                "param1": {"type": "string", "description": "参数说明"}
            }
        }
    }
]
```

### call_tool(tool_name, input_data)

执行指定工具。返回纯 dict（可 JSON 序列化）：

```json
// 成功
{"result": "...", "status": "success"}

// 失败
{"error": "Tool 'xxx' not found."}
```

### get_metadata()（v2.0+）

返回 Server 元数据：

```json
{
    "name": "MyServer",
    "version": "1.0.0",
    "category": "structural",
    "description": "一句话说明",
    "tools": [{"name": "...", "description": "..."}],
    "dependencies": [],
    "compatibility": {"caiao_spec": "1.0", "mcp": true}
}
```

## 规则列表

| 规则 | 说明 |
|------|------|
| **R1** | 始终返回 `[TextContent(type="text", text=json.dumps(...))]`（MCP 版本）或纯 dict（轻量版本）——绝不返回原始字符串 |
| **R2** | 所有内容序列化为 JSON——Gateway 从 JSON 字符串中提取 result |
| **R3** | 捕获所有异常并返回 `{"error": str(e)}`——绝不崩溃子进程 |
| **R4** | 工具名使用 snake_case——与 Python 惯例一致 |
| **R5** | 输入 Schema 是 JSON Schema——LLM 用其生成正确参数 |
| **R6** | 非惰性 Server 必须快速启动——模块级别不做重量级导入 |
| **R7** | 惰性 Server 必须优雅降级——依赖缺失时返回 `{"error": "unavailable"}` |

## 输入 Schema 设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **自描述** | 每个字段有 description，AI 可无歧义理解 |
| 2 | **最小完备** | 只含必要字段，不冗余 |
| 3 | **类型严格** | 物理量用 number，索引用 integer，ID 用 string |
| 4 | **可扩展** | 不设 `additionalProperties: false`，允许传额外字段 |
| 5 | **声明约束** | 用 minimum、maximum、enum、pattern 等约束声明 |

## 输出规范

工具的返回值必须是纯 dict，所有值可 JSON 序列化：

正确示例：返回包含 status、data、calc_processes 等字段的 dict。
错误示例：直接返回 NumPy 数组（应用 .tolist() 转换）、返回自定义类实例。

对于需要展示详细计算步骤的场景，在输出中附加 `calc_processes` 数组，每步包含公式、参数值、结果、单位。
