# caiao.yaml 清单系统规范

## 概述

`caiao.yaml` 是每个 CAIAO Server 的声明式清单文件。Gateway 通过 `caiao/discovery.py` 自动扫描 `servers/*/caiao.yaml`，转换为 Hub 可用的 SERVER_CONFIGS 条目。

## 文件位置

```
servers/<server_name>/caiao.yaml
```

文件名固定为 `caiao.yaml`，与 `server.py` 同目录。

## 完整字段参考

### 身份（Identity）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | string | 是 | Server 唯一名称，通常与目录名一致 |
| `version` | string | 是 | 语义化版本，如 `0.1.0` |
| `kind` | string | 是 | `atomic-mcp` / `composite` / `infrastructure` |
| `description` | string | 是 | 一句话用途说明 |
| `status` | string | 否 | `active` / `deprecated` / `experimental` / `maintenance` |
| `since` | string | 否 | 创建日期，ISO 格式 |

### 生命周期（Lifecycle）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `start_mode` | string | 是 | `eager`（gateway 启动时 spawn）/ `lazy`（首次调用时 spawn） |

Composite Server 不需要此字段。

### 命令（Command）— 仅 subprocess Server

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `command.python` | string | 是 | Python 路径。`auto` = 使用 gateway venv 的 Python。`@abaqus_python@` = 从 abaqus_env.json 解析 |
| `command.args` | list | 是 | 命令行参数，通常 `["server.py"]` |
| `command.cwd` | string | 是 | 工作目录，相对于清单文件所在目录 |
| `command.env` | dict | 否 | 额外环境变量 |

### 健康（Health）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `health.timeout_ms` | int | 否 | 启动超时（毫秒），默认 5000 |
| `health.restart_on_crash` | bool | 否 | 崩溃时是否自动重启，默认 false |
| `health.max_restarts` | int | 否 | 最大重启次数，默认 3 |
| `health.health_check_interval_s` | int | 否 | 定期健康检查间隔（秒），0 表示不检查 |

### 工具（Tools）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `tools[].name` | string | 是 | 工具名称，snake_case |
| `tools[].description` | string | 是 | 工具描述，AI 据此理解功能 |
| `tools[].tags` | list | 否 | 分类标签，用于语义搜索 |

### 管线（Pipeline）— 仅 composite Server

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `pipeline[].server` | string | 是 | 目标 Server 名称 |
| `pipeline[].tool` | string | 是 | 要调用的工具名 |
| `pipeline[].map_result` | string | 否 | 将结果存储为上下文的变量名 |
| `pipeline[].input_map` | dict | 否 | 从上下文映射到工具参数 |

### 导入（Imports）— 仅 merged Server

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `imports[].module` | string | 是 | 源模块路径 |
| `imports[].symbols` | list | 是 | 要导入的符号名 |

### 依赖（Dependencies）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `dependencies.python` | list | 否 | Python 包依赖，如 `["numpy", "pyyaml"]` |
| `dependencies.system` | list | 否 | 系统依赖，如 `["blender>=4.2"]` |
| `depends_on.servers` | list | 否 | 依赖的其他 CAIAO Server 名（用于 infrastructure 模式） |

### 能力（Capabilities）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `capabilities` | list | 否 | 能力标签列表，用于语义搜索和分类 |

### 测试（Testing）

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `testing.test_file` | string | 否 | 测试文件名 |
| `testing.test_command` | list | 否 | 测试运行命令 |

## 最小清单示例（atomic-mcp）

```yaml
name: my_solver
version: 0.1.0
kind: atomic-mcp
description: "一句话说明这个 Server 做什么"
start_mode: lazy
command:
  python: auto
  args: ["server.py"]
  cwd: "."
tools:
  - name: my_tool
    description: "工具描述，AI 据此理解何时调用"
    tags: [demo]
```

## Composite 清单示例

```yaml
name: my_pipeline
kind: composite
description: "编排 A → B → C"
tools:
  - name: run_my_pipeline
    description: "执行完整管线"
pipeline:
  - server: server_a
    tool: tool_a
    map_result: result_a
  - server: server_b
    tool: tool_b
    input_map:
      input_field: result_a
    map_result: result_b
```

## Infrastructure 清单示例

```yaml
name: my_environment_server
kind: infrastructure
description: "提供共享运行时环境"
start_mode: eager
command:
  python: auto
  args: ["server.py"]
  cwd: "."
tools:
  - name: resolve_runtime_path
    description: "发现运行时路径"
    tags: [environment, discovery]
  - name: validate_environment
    description: "全面环境健康检查"
    tags: [diagnostics, validation]
```

## 自动发现机制

`caiao/discovery.py` 中的 `discover_server_configs()` 函数：

1. 遍历 `servers/` 下所有子目录（跳过 `_` 和 `.` 开头的）
2. 读取每个目录下的 `caiao.yaml`
3. 通过 `_manifest_to_config()` 转换为 Hub 可用的配置字典
4. 处理 sentinel 值（如 `@abaqus_python@` 从 `abaqus_env.json` 动态解析）
5. 未找到清单的目录回退到 legacy 硬编码配置
