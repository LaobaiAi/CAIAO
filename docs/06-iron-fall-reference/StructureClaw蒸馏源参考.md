# 从外部项目蒸馏领域能力的方法论

> 基于从 StructureClaw（一个 AI 辅助结构工程开源项目）蒸馏能力的实践经验。
> 详细蒸馏操作见 `../05-dev-guides/蒸馏方法论.md`。

---

## 一、什么是"蒸馏"

在 CAIAO 体系中，"蒸馏"指从一个复杂的现有系统中提取领域计算能力，封装为独立、零依赖的 CAIAO Server 的过程。

蒸馏 vs 迁移的区别：
- **迁移**：将整个求解器封装为 Server，保留原算法和逻辑
- **蒸馏**：理解源系统的领域知识后，从零重新实现，不引入源系统的代码依赖

---

## 二、蒸馏前的评估

在决定蒸馏一个外部项目之前，先回答：

1. 源系统的领域边界是什么？它解决了什么问题？
2. 源系统的核心算法是否公开、可理解、可独立复现？
3. 源系统的技术栈是否与目标环境兼容？是否需要隔离？
4. 蒸馏后能否做到零依赖（或仅依赖标准库）？

---

## 三、关键取舍原则

蒸馏过程中需要做的典型决策：

| 决策点 | 取舍考量 |
|--------|---------|
| 核心算法 | 优先选择可自实现的标准算法，零安装门槛优于功能完整 |
| 数据管理 | 深度有限时硬编码内置数据优于外部文件依赖 |
| 通信模式 | 开发效率优先时进程内直调优于子进程隔离 |
| 输出格式 | 环境受限时模板 + 字符串 fallback 优于重量模板引擎 |
| 依赖管理 | 可选依赖不可用时自动降级，不阻塞核心功能 |

---

## 四、蒸馏经验

1. **先理解后实现**：花足够时间理解源系统的数据流和算法粒度后再动手，避免照搬和误读
2. **简化有边界**：核心计算（如数学/物理模型）可参考教材标准实现，不自行创造
3. **降级方案内置**：可选依赖不可用时自动 fallback，不阻断主流程
4. **测试先行**：用源系统的输入输出作为测试用例的参考值，验证数值一致性
